## IV. Distribution of the Day-30 retrospective

- **Internal:** full retrospective shared with team, GK, exec
- **Investors / strategic partners:** abbreviated version (top 3 wins, top 3 misses, next 3 priorities) in next investor update
- **Public:** never the full retrospective. A blog post summarizing top 1-2 wins + Phase-2 outlook only.

---

## V. Decision triggers from the retrospective

If we are…

- **At or above target on most metrics:** continue current cadence, double down on what worked
- **Between floor and target:** continue cadence, iterate on the weakest assets
- **Below floor on critical metrics (pickup, sentiment):** pause press cadence, regroup on positioning before next push
- **Below floor on community / signup metrics:** issue is funnel / asset quality, not story — fix the funnel
- **Below floor on developer metrics:** docs / Quickstart issue — engineering priority

---

## VI. Critical metrics that override everything

If any of these happen, the retrospective gets superseded by a different process:

- **Regulatory inquiry surfaces** → switch to legal / regulatory response playbook
- **Hostile coverage from major outlet** → crisis comms (`23-crisis-pushback-playbook.md`)
- **Significant token dump pre-TGE** → token + community comms emergency
- **UXLINK pulls or distances** → narrative reset required
- **Major security incident in Astarter products** → incident response over comms

In any of these cases, the retrospective is paused and the appropriate playbook takes priority.
